# AI-DLC Metrics Tracker

This file is the quantitative record of how AI-DLC performed on this project, Bolt by Bolt. These numbers feed directly into the public case study and are the evidence behind any productivity claim on the CV.

## How to Fill This Out

At the end of every Bolt, update its row before merging. Do not backfill from memory — it's always wrong. Keep a rough running count during the Bolt in a scratch file and copy the totals here.

### Column definitions

- **Clock hours** — wall-clock time actively spent on the Bolt (Elaboration + Construction + Gate Evidence + retro). Do not count time the app was in the background.
- **Prompts** — total number of distinct prompts sent to Claude Code for this Bolt.
- **LOC generated** — lines of code generated by the AI and merged (approximate, via git diff --stat on the Bolt branch).
- **LOC hand-written** — lines the developer wrote or substantively modified by hand.
- **Tests added** — count of automated tests added during the Bolt.
- **Gate failures** — number of times a Gate Evidence item failed the first time and needed rework.
- **Production bugs found later** — incremented when a bug from this Bolt is discovered in a later Bolt.
